Type: Object

Properties

blue (number) : The blue output strength, 0-255 inclusive.

UfoDiscoverOptions

Ufo.discover options.

UfoDiscoverOptions

Type: Object

Properties
timeout (number?) : How long to wait for UFOs to respond, in milliseconds. Default is 3000.
password (string?) : The UDP password to use when searching for UFOs. If unspecified, the default password is used.
remotePort (number?) : The UDP port to which expected UFOs are bound. If unspecified, the default port 48899 is used.
localPort (number?) : The UDP port bound on this machine to perform the UFO search. If unspecified, a random port is used.
localAddress (string?) : The local host used for establishing the UDP socket.

WifiNetwork

Type: Object

Properties

security (string) : A description of the security configuration for this network.
strength (number) : The network signal strength, 0-100 inclusive.

▸ setBuiltin(name, speed)

Starts one of the UFO's built-in functions at the given speed. The promise will be rejected if an invalid function name is given.

The speed is clamped from 0-100 inclusive. Speed values do not result in the same durations across all functions (e.g. sevenColorStrobeFlash is much faster at speed 100 than sevenColorJumpingChange); you will need to experiment with different values to get the desired timing for the function you wish to use.

▸ setCustom(mode, speed, steps)

Starts the given custom function. The promise will be rejected if an invalid mode is given.

  • The speed is clamped from 0-30 inclusive. Below is a list of step durations measured with a stopwatch when using the "jumping" mode. These values should be treated as approximations. Based on this list, it appears decrementing the speed by 1 increases step duration by 0.14 seconds.

    • 30 = 0.4 seconds
    • 25 = 1.1 seconds
    • 20 = 1.8 seconds
    • 15 = 2.5 seconds
    • 10 = 3.2 seconds
    • 5 = 3.9 seconds
    • 0 = 4.6 seconds
  • Only the first 16 steps in the given array are considered. Any additional steps are ignored.
  • If any null steps are specified in the array, they are dropped before the limit of 16 documented above is considered.

▸ getWifiAutoSwitch()

Returns the UFO's WiFi "auto-switch" setting, which is one of the following:

  • "off", which means auto-switch is disabled.
  • "on", which means 1 minutes.
  • "auto", which means 10 minutes.
  • The number of minutes, 3-120 inclusive.

If auto-switch is anything but "off", and if the UFO fails to connect to its client AP or otherwise enters any abnormal state, the UFO will reset itself and enable its AP mode after the specified number of minutes have passed.

Note that this method always returns a string even if the value is a number.

▸ getWifiClientAuth()

Returns the UFO client's AP auth settings.

  • If auth is OPEN, encryption is NONE, WEP-H or WEP-A.
  • If auth is SHARED, encryption is WEP-H or WEP-A.
  • If auth is WPAPSK or WPA2PSK, encryption is TKIP or AES.
  • If encryption is NONE, paraphrase is null.
  • If encryption is WEP-H, paraphrase is exactly 10 or 26 hexadecimal characters in length.
  • If encryption is WEP-A, paraphrase is exactly 5 or 13 ASCII characters in length.
  • If encryption is TKIP or AES, paraphrase is 8-63 ASCII characters in length, inclusive.

UfoOptions

Available configuration options for the Ufo object.

UfoOptions

Type: Object

Properties
host (string) : The IP address of the UFO. If you want a fixed IP address, you can either configure static DHCP assignment on your router using the UFO's MAC address or you can use the CLI to configure the UFO's WiFi client settings to use a static IP instead of DHCP.
password (string?) : the UDP password used to connect to the UFO. If unspecified, the default is used. You can use the CLI to change the UFO's password for greater security, since the default password is well-known and hardcoded into this library.
localHost (string?) : The local host used for establishing the TCP socket.
localUdpPort (number?) : the port number that will be bound on this machine for UDP traffic to the UFO. If unspecified, a random port is used.
remoteUdpPort (number?) : the UDP port number on the UFO. If unspecified, the default port 48899 is used. You can change this using the CLI.
localTcpPort (number?) : the port number that will be bound on this machine for UDP traffic to the UFO. If unspecified, a random port is used.
remoteTcpPort (number?) : the UDP port number on the UFO. If unspecified, the default port 5577 is used. You can change this using the CLI.
immediate (boolean?) : if true or unspecified, socket.setNoDelay(true) is invoked when the TCP socket is created. This prevents TCP data from being buffered by NodeJS before being sent to the UFO. If this causes communication issues, set this to false.
cache (boolean?) : if true or unspecified, the UFO object will cache the UFO's status internally. This speeds up future calls to methods that return/make use of the UFO's status. This cache is used only when the last known UFO mode is "static"; the cache is invalidated whenever a builtin or custom function is invoked on the UFO.
